# Visual Design Build

## Purpose
This document defines how the website UI will be designed before any code is written.

The site will be planned visually first through storyboards, page structure decisions, and mobile-first layout direction. Development should begin only after the visual plan for each page is approved.

## Goal
Create a complete visual plan for the website before implementation begins.

This process should:
- establish the visual direction of the site
- define the structure and purpose of each page
- clarify CTA placement and conversion flow
- align messaging and trust-building elements
- make implementation easier once design decisions are approved

## Process
The visual design build will happen one page at a time.

Each conversation should focus on one page only.

Each page conversation should end with an approved storyboard for that page, including:
- page purpose
- visitor intent
- primary CTA
- secondary CTA
- section order
- messaging hierarchy
- trust-building elements
- mobile-first layout behavior
- visual style direction
- content or asset needs

No code should be written during this phase.

## Conversation Plan
The site should be designed across a sequence of focused conversations:

1. Homepage
2. Sellers Page
3. About Eric Page
4. Contact Page
5. Thank-You Page
6. Search Homes / IDX Page
7. Shared mobile UI patterns and reusable sections
8. Final full-site visual consistency review

## Why This Order
The homepage should be designed first because it sets the overall visual language for the site.

It should define:
- seller-focused messaging direction
- CTA style
- trust signals
- section rhythm
- mobile conversion behavior

The Thank-You page should be designed before IDX because it is part of the core seller conversion funnel.

## Standard Structure For Each Page Conversation
Each page conversation should follow the same structure:

1. Page purpose
2. User mindset when landing on the page
3. Primary CTA
4. Secondary CTA
5. Section-by-section storyboard
6. Mobile-first behavior
7. Trust-building elements
8. Content and asset needs
9. Final approved page brief

## Required Output For Each Page
Each page conversation should end with:
- a page brief
- a section-by-section storyboard
- mobile layout notes
- CTA placement plan
- trust and branding notes
- asset and content checklist
- final approval summary

## Design Priorities
All visual design conversations should follow these priorities:

1. Seller lead generation comes first
2. Mobile-first layout decisions are required
3. Trust-building should be visible throughout the experience
4. Shared design patterns should be reused across pages
5. Unresolved questions should be documented clearly for later decisions

## Working Rules
- One page per conversation
- No coding during the storyboard phase
- Keep seller conversion as the primary focus
- Treat IDX as a secondary experience
- Reuse approved patterns across the site where possible
- Capture unresolved decisions instead of improvising them

## Expected Deliverables
By the end of the visual design build, the project should have:
- an approved storyboard for every core page
- a shared visual direction for the full site
- a list of reusable UI patterns
- a page-by-page implementation-ready design plan

## Recommended Starting Point
Start with the homepage.

The homepage will establish the visual foundation for:
- branding tone
- page rhythm
- CTA placement
- testimonial treatment
- trust-building patterns
- mobile sticky actions

Once the homepage storyboard is approved, the remaining pages should follow the same system.
